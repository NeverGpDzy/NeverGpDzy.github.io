Docker ж—Ҙеёёе‘Ҫд»ӨйҖҹжҹҘ

иҝҷйЎөеҒҸйҖҹжҹҘпјҢдёҚиҝҪжұӮжҰӮеҝөе®Ңж•ҙпјҢдё»иҰҒжҳҜжҠҠжҲ‘жңҖеёёеӣһзңӢзҡ„е‘Ҫд»Өж”ҫеңЁдёҖиө·гҖӮй»ҳи®ӨдҪҝз”Ёиҫғж–°зҡ„еҜ№иұЎејҸеҶҷжі•пјҢдҫӢеҰӮ docker image lsгҖҒdocker container lsпјҢдҪҶжҲ‘д№ҹдҝқз•ҷдәҶеёёи§ҒеҲ«еҗҚжҖқи·ҜпјҢйҒҝе…ҚжҹҘе‘Ҫд»Өж—¶иҝҳиҰҒеҒҡдәҢж¬Ўзҝ»иҜ‘гҖӮ

е…Ҳи®°дҪҸжңҖе°ҸжЁЎеһӢвҖӢ

й•ңеғҸ image -> docker run -> е®№еҷЁ container
  • й•ңеғҸеғҸжЁЎжқҝ
  • е®№еҷЁеғҸиҝҗиЎҢдёӯзҡ„е®һдҫӢ
  • еҲ йҷӨе®№еҷЁйҖҡеёёдёҚдјҡеҲ й•ңеғҸ
  • ж•°жҚ®жҢҒд№…еҢ–иҰҒйқ  volume жҲ– bind mountпјҢдёҚиҰҒдҫқиө–е®№еҷЁеҸҜеҶҷеұӮ

й•ңеғҸзӣёе…івҖӢ

зӣ®зҡ„е‘Ҫд»Ө
жҹҘзңӢжң¬ең°й•ңеғҸdocker image ls
жӢүеҸ–й•ңеғҸdocker pull nginx:stable
жҹҘзңӢй•ңеғҸиҜҰжғ…docker image inspect nginx:stable
з»ҷй•ңеғҸжү“ж Үзӯҫdocker tag nginx:stable my-nginx:demo
жҹҘзңӢй•ңеғҸжһ„е»әеҺҶеҸІdocker history nginx:stable
еҲ йҷӨй•ңеғҸdocker image rm my-nginx:demo

еёёи§ҒеҲ«еҗҚпјҡ

  • docker images зӯүд»·дәҺ docker image ls

е®№еҷЁзӣёе…івҖӢ

зӣ®зҡ„е‘Ҫд»Ө
жҹҘзңӢиҝҗиЎҢдёӯзҡ„е®№еҷЁdocker container ls
жҹҘзңӢжүҖжңүе®№еҷЁdocker container ls -a
еҲӣе»әе№¶иҝҗиЎҢе®№еҷЁdocker run -d --name web -p 8080:80 nginx:stable
еҗҜеҠЁе·ІеӯҳеңЁе®№еҷЁdocker start web
еҒңжӯўе®№еҷЁdocker stop web
ејәеҲ¶жқҖжҺүе®№еҷЁdocker kill web
йҮҚеҗҜе®№еҷЁdocker restart web
еҲ йҷӨе®№еҷЁdocker rm web
ејәеҲ¶еҲ йҷӨиҝҗиЎҢдёӯзҡ„е®№еҷЁdocker rm -f web
дҝ®ж”№е®№еҷЁеҗҚdocker rename old-name new-name

еёёи§ҒеҲ«еҗҚпјҡ

  • docker ps зӯүд»·дәҺ docker container ls
  • docker ps -a зӯүд»·дәҺ docker container ls -a

иҝӣе…Ҙе®№еҷЁе’ҢжӢ·иҙқж–Үд»¶вҖӢ

иҝӣе…ҘжӯЈеңЁиҝҗиЎҢзҡ„е®№еҷЁвҖӢ

docker exec -it web bash

еҰӮжһңй•ңеғҸйҮҢжІЎжңү bashпјҢе°ұз”Ё shпјҡ

docker exec -it web sh

д»Ҙ root иә«д»Ҫиҝӣе…Ҙе®№еҷЁвҖӢ

docker exec -u root -it web bash

дё»жңәе’Ңе®№еҷЁд№Ӣй—ҙеӨҚеҲ¶ж–Үд»¶вҖӢ

docker cp ./local.txt web:/tmp/local.txt
docker cp web:/var/log/nginx/access.log ./access.log

ж—Ҙеҝ—гҖҒзҠ¶жҖҒе’ҢжЈҖжҹҘвҖӢ

зӣ®зҡ„е‘Ҫд»Ө
жҹҘзңӢе®№еҷЁж—Ҙеҝ—docker logs web
жҢҒз»ӯи·ҹж—Ҙеҝ—docker logs -f --tail 100 web
зңӢе®№еҷЁиҜҰз»ҶдҝЎжҒҜdocker inspect web
еҸӘзңӢйҖҖеҮәз Ғdocker inspect --format '{{.State.ExitCode}}' web
зңӢе®№еҷЁиҝӣзЁӢdocker top web
зңӢиө„жәҗдҪҝз”Ёdocker stats
зңӢз«ҜеҸЈжҳ е°„docker port web

еҰӮжһңдҪ дёҚзҹҘйҒ“е®№еҷЁдёәд»Җд№ҲйҖҖеҮәпјҢдјҳе…ҲжҢүиҝҷдёӘйЎәеәҸжҹҘпјҡ

  1. docker ps -a
  2. docker logs <container>
  3. docker inspect <container>

зҪ‘з»ңе’ҢеӯҳеӮЁвҖӢ

docker network ls
docker volume ls
docker volume inspect mydata

еҰӮжһңдҪ з»ҸеёёеңЁвҖңж•°жҚ®еҺ»е“ӘдәҶвҖқвҖңжҢӮиҪҪдёәд»Җд№ҲдёҚеҜ№вҖқд№Ӣй—ҙеҸҚеӨҚи·іпјҢзӣҙжҺҘзңӢпјҡ

  • Docker ж•°жҚ®жҢҒд№…еҢ–дёҺжҢӮиҪҪ
  • Docker з«ҜеҸЈжҳ е°„дёҺи®ҝй—®иҢғеӣҙжҺ§еҲ¶

жё…зҗҶе‘Ҫд»ӨвҖӢ

е…ҲзңӢзЈҒзӣҳеҚ з”Ёпјҡ

docker system df

еҶҚжҢүеҜ№иұЎжё…зҗҶпјҡ

docker container prune
docker image prune
docker volume prune
docker network prune

еҰӮжһңдҪ йқһеёёзЎ®е®ҡиҰҒеҒҡдёҖж¬ЎеӨ§жү«йҷӨпјҡ

docker system prune -a --volumes

иҝҷдёӘе‘Ҫд»ӨдјҡеҲ жҺүжңӘдҪҝз”Ёзҡ„е®№еҷЁгҖҒзҪ‘з»ңгҖҒй•ңеғҸе’Ңж•°жҚ®еҚ·гҖӮе®ғеҫҲзңҒдәӢпјҢдҪҶд№ҹеҫҲе®№жҳ“жҠҠвҖңжҡӮж—¶жІЎеңЁиҝҗиЎҢгҖҒдҪҶдҪ иҝҳжғіз•ҷзқҖвҖқзҡ„дёңиҘҝдёҖиө·жё…жҺүгҖӮ

жҲ‘иҮӘе·ұжңҖеёёз”Ёзҡ„дёҖз»„з»„еҗҲвҖӢ

docker ps
docker logs -f --tail 100 my-service
docker exec -it my-service bash
docker inspect my-service
docker system df
