网站上线检查清单

真正把网站发出去之前，最好做一次完整检查。

这个清单不追求覆盖所有高级场景，而是先把第一次上线最常漏掉的地方挡住。

1. 可访问性检查​

  • 首页能访问
  • 深层页面能直接访问
  • 裸域和 www 行为符合预期
  • HTTP 到 HTTPS 重定向符合预期
  • 移动端和桌面端都能正常打开

2. 域名与 DNS 检查​

  • 解析记录指向正确
  • 权威 DNS 服务商明确
  • Nameserver 是否已经切对
  • 变更后已经完成基本验证

3. HTTPS 检查​

  • 证书有效
  • 域名覆盖完整
  • 没有明显混合内容
  • 重定向链不过长

4. 内容与路径检查​

  • 主要导航可达
  • 关键文档没有 404
  • 图片、样式、脚本路径正常
  • 常用入口页已经建立

5. 缓存与资源检查​

  • HTML 和静态资源缓存策略已区分
  • 大图片已压缩
  • 明显可长期缓存的资源已有版本化策略
  • CDN 接入后没有出现旧内容迟迟不更新

6. 搜索与基础 SEO 检查​

  • 页面标题基本准确
  • robots.txt 没有低级误配
  • sitemap 可用
  • 主要文档之间有内部链接

7. 观察与回滚准备​

  • 至少有基本访问分析或日志
  • 知道构建产物从哪来
  • 知道发布失败时怎么回滚到上一个版本
  • 知道改 DNS / 缓存后怎么验证

8. 服务器路线额外检查​

如果你走的是服务器部署，再额外检查：

  • Nginx 配置是否生效
  • 应用进程是否稳定
  • 端口暴露是否符合预期
  • 日志是否可查看
  • 重启后服务是否能恢复

这份清单的目的​

不是让你一次做到完美，而是避免这些典型错误：

  • 域名能开但证书错
  • 首页能开但深链 404
  • 页面能开但资源全丢
  • 接了 CDN 但缓存策略混乱
  • 已经公开发布却完全不知道站点状态

适合接着回看什么​

  • DNS 问题：看 域名、DNS 与解析链路
  • HTTPS 问题：看 HTTP 与 HTTPS
  • CDN / 缓存问题：看 CDN 原理与应用
  • 性能问题：看 Web 性能基础：从首屏到缓存策略
