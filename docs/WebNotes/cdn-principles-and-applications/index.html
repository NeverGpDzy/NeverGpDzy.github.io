CDN 原理与应用

CDN 是网站性能优化里最常见、也最容易被先接触到的概念之一。理解它最重要的不是背定义，而是知道它到底解决了什么问题、适合缓存什么内容、又该怎么配置。

什么是 CDN​

CDN（Content Delivery Network，内容分发网络）是一种分布式服务器系统。它通过把内容缓存到全球各地的节点服务器上，加速内容交付，提升网站加载速度和可用性，同时减轻源站压力。

CDN 如何工作​

  1. 缓存内容：将图片、CSS、JavaScript 等静态内容缓存到各地节点。
  2. 就近分发：用户请求内容时，优先由距离更近的节点响应，降低延迟。
  3. 动态加速：对于不适合长时间缓存的动态内容，也可以通过优化传输路径来提速。

适合使用 CDN 的内容类型​

静态内容​

  • 图像文件：JPEG、PNG、GIF、SVG、WebP 等
  • 视频和音频文件：MP4、WebM、Ogg、MP3、AAC 等
  • 文档和下载文件：PDF、DOCX、PPTX、TXT、ZIP、RAR 等
  • 网页资源文件：HTML、CSS、JavaScript
  • 字体文件：TTF、OTF、WOFF、WOFF2
  • 数据文件：JSON、XML、CSV
  • 图形和动画资源：SVG、Canvas
  • 3D 模型和资产：OBJ、FBX、GLTF
  • 静态网站生成器输出内容

静态内容最适合走 CDN，因为缓存命中后，节点可以直接返回资源，源站基本不需要额外参与。

静态内容接入 CDN 的常见步骤​

1. 选择 CDN 提供商​

常见方案包括 Cloudflare、Akamai、Amazon CloudFront、Google Cloud CDN 等。

2. 配置 DNS​

把你的域名解析指向 CDN 提供商指定的地址。常见形式是添加一条 CNAME 记录：

www.example.com CNAME cdn.example-cdn.com

3. 配置 CDN 规则​

在 CDN 控制台中配置哪些路径需要缓存、缓存多久、哪些资源要回源。

4. 检查源站​

确认源站允许 CDN 节点访问静态资源，并且返回合理的缓存头。

5. 测试缓存命中​

通过浏览器开发者工具或响应头检查资源是否真的经由 CDN 返回。

6. 配置缓存策略​

根据业务需要设置缓存时长、刷新策略和版本控制方式。

一个典型的静态资源请求流程​

  1. 用户请求 https://www.example.com/index.html
  2. CDN 节点检查本地缓存
  3. 如果命中，节点直接返回缓存内容
  4. 如果未命中，节点回源获取内容并缓存，供后续请求复用

常见响应头示例​

HTTP/1.1 200 OK
Date: Sat, 08 Jun 2024 10:00:00 GMT
Content-Type: text/html
Content-Length: 1024
Connection: keep-alive
Cache-Control: max-age=3600
CDN-Cache: HIT
Server: example-cdn

CDN 如何处理动态内容​

动态内容的典型例子​

  • 用户资料、订单历史、消息通知
  • 新闻流、社交动态、股票价格、天气预报
  • 推荐结果、个性化广告
  • 搜索结果页
  • 购物车和结账页面

这类内容往往不能像静态资源那样长期缓存，但仍然可以通过合理的缓存头和边缘逻辑做优化。

常见优化方式​

  • 缓存控制头：用 Cache-Control、Expires、ETag、Last-Modified 控制缓存行为
  • Vary 头：根据请求头区分缓存版本
  • 边缘计算：在 CDN 节点运行自定义脚本，例如 Cloudflare Workers、AWS Lambda@Edge
  • 路径规则：明确哪些路径缓存、哪些路径不缓存
  • 短期缓存：对部分动态内容设置很短的缓存时间，减轻源站压力

配置示例​

# 针对静态内容
Cache-Control: public, max-age=31536000

# 针对动态内容（短期缓存）
Cache-Control: private, max-age=60

# 针对不缓存的 API 请求
Cache-Control: no-store

总结​

CDN 的核心不是“神秘地加速”，而是把适合缓存的内容提前分发到离用户更近的地方，同时让不适合缓存的内容也尽可能更高效地传输。理解缓存命中、回源、缓存策略这几个关键词，基本就抓住了 CDN 的主线。
